# Using the Next Web Artifacts Builder skill

This document provides a few pointers for developing artifacts with the
Next.js template generated by this skill.

## Running the development server

After running the `init-artifact.sh` script and changing into your new
project directory, start the development server:

```bash
pnpm run dev
```

Open the printed local URL in your browser to see the live site. As you edit
files under `app/` or `components/`, the page will hot‑reload.

## Creating new pages

The template uses the **Next.js app router**. To add a new page, create a
folder inside `app/` with an `page.tsx` file. For example, to add an "About"
page:

```bash
mkdir app/about
cat > app/about/page.tsx <<'TS'
export default function About() {
  return (
    <div className="p-8">
      <h1 className="title">About this artifact</h1>
      <p className="paragraph mt-4">You can create as many pages as you like. Each
        page file inside the `app/` directory becomes a route.</p>
    </div>
  )
}
TS
```

Next.js automatically code‑splits and lazy‑loads these pages.

## Using shadcn/ui components

This template installs all peer dependencies required by the shadcn/ui component
library but does not include prebuilt components. You can add components on
demand using the official CLI:

```bash
npx shadcn-ui@latest add button
```

Follow the prompts to choose your preferred style variant. The CLI will create
the component under `components/ui/` and update your Tailwind configuration
accordingly. You can repeat this process for any shadcn/ui component listed in
the [documentation](https://ui.shadcn.com/docs/components).

## Dark mode support

The template uses [next-themes](https://github.com/pacocoursey/next-themes) for
dark mode. To allow users to switch themes, wrap your layout with the
`ThemeProvider` from `next-themes` and add a button or toggle that calls
`setTheme('light')` or `setTheme('dark')`. See the next-themes documentation
for details.

## Bundling your artifact

When your project is ready to share, run:

```bash
bash ../scripts/bundle-artifact.sh
```

This will generate a `bundle.html` file in your project root. The file is
self‑contained and includes all of the CSS and JavaScript needed to run your
artifact offline. Attach this file in your Claude conversation to share your
creation.
