The backend Dockerfile has no USER instruction before the final ENTRYPOINT. If an attacker controls any process inside the container, they have root access to the container (and potentially the host if not properly namespaced).
backend/Dockerfile:37 · Rule: missing-user-entrypoint

04 · OWASP ZAP Skill
OWASP Risk Assessment (Static — App Not Running)
ZAP Dynamic Scan — Not Run

The OWASP ZAP skill requires the application running on a port. Command: docker run --rm zaproxy/zap-stable zap-full-scan.py -t http://host.docker.internal:3001
Run this once the app is live and save output to owasp_zap_report.txt per the skill instructions. The findings below are based on code analysis and Semgrep results.

A01:2021 — Broken Access Control
Path Traversal in File Serving
execute.controller.ts serves files using user-supplied runId and filename directly in path.join() without adequate canonicalization. An attacker can escape the run directory and read arbitrary files.
Confirmed by Semgrep: execute.controller.ts:145,153 · docker-executor.service.ts:123,316,341
A04:2021 — Insecure Design
Docker Container Runs as Root
Backend Dockerfile has no USER directive. The NestJS process runs as root inside the container. Combined with path traversal in the execute module, this dramatically increases blast radius if exploited.
Confirmed by Semgrep: backend/Dockerfile:37 · CWE-269
A06:2021 — Vulnerable & Outdated Components
multer DoS · jspdf Injection · axios DoS
multer (file upload) has 3 DoS CVEs. jspdf has PDF injection allowing arbitrary JS execution. axios has DoS via prototype pollution. All have available fixes.
npm audit: backend 9 HIGH, frontend 3 HIGH — all fixable
A07:2021 — Identification & Auth Failures
No OAuth Token Refresh Handling
The Codebasics access token expires in 15 days (per skill spec). auth.ts stores expiresAt in the JWT but no refresh logic exists. Sessions with expired OAuth tokens will silently fail API calls without re-auth prompts.
auth.ts:32 stores expiresAt but no refresh callback implemented
A05:2021 — Security Misconfiguration
Missing Security Headers
No Content-Security-Policy, X-Frame-Options, X-Content-Type-Options, or HSTS headers configured in NestJS or Next.js. ZAP scan will flag these. Monaco editor renders user code — CSP is important here.
No helmet.js or Next.js headers config found in next.config.js
A09:2021 — Security Logging & Monitoring
No Error Tracking / Security Audit Log
No Sentry, no structured security logging for auth failures or admin actions. Code execution errors are returned to the client but not logged for security monitoring. Admin CRUD actions have no audit trail.
No sentry SDK in package.json · no winston security transport configured

06 · Remediation Priority
What to Fix, in Order
P1
Patch path traversal in execute.controller.ts and questions.service.ts
Validate runId as UUID, restrict filename to /^[\w.-]+\.(png|csv|json)$/, canonicalize with path.resolve() and assert result starts with the known base directory before serving. This is a pre-auth file read risk (CWE-22, OWASP A01).
P1
Add USER to backend Dockerfile
One-line fix: add RUN addgroup -S app && adduser -S app -G app followed by USER app before the CMD. Combined with path traversal, running as root is a severe escalation path.
P2
Run npm audit fix on both frontend and backend
Update axios, jspdf, multer, @monaco-editor/react. Most fixes are non-breaking. The prisma → 6.x upgrade is semver major — test separately. Address the jspdf injection (CWE-94) before PDF export goes to production users.
P2
Fix brand compliance in globals.css and layout.tsx
Replace Inter with Saira Condensed + Kanit. Replace #f8fafc with #181830. Replace #6366f1 with #3B82F6. Register all brand color tokens in Tailwind config. This is the most visible deviation from the brand guidelines skill — every learner sees it.
P3
Add security headers (helmet.js in NestJS · headers config in next.config.js)
Configure CSP, X-Frame-Options, X-Content-Type-Options, HSTS. Especially important for Monaco Editor (renders code) and the PDF export feature. OWASP ZAP will flag these as medium-severity findings.
P3
Fix OAuth token expiry handling + JWT type declarations
Add expiry check in the jwt callback. Fix next-auth.d.ts to declare the actual stored fields (userId, userRole, etc.) instead of the unused codebasicsProfile. Add an error page for OAuth failures.
P4
Run OWASP ZAP with the app live, save to owasp_zap_report.txt
Start the app, then: docker run --rm zaproxy/zap-stable zap-full-scan.py -t http://host.docker.internal:3001 2>&1 | tee owasp_zap_report.txt. Repeat for port 3000 (frontend). Fix any additional findings surfaced by the dynamic scan.
